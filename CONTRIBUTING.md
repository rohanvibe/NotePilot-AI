# Contributing to NotePilot AI

First off, thank you for considering contributing to NotePilot AI! It's people like you that make NotePilot such a great tool.

## How Can I Contribute?

### Reporting Bugs
* Check the existing issues to see if the bug has already been reported.
* If you can't find an open issue addressing the problem, open a new one.
* Use a clear and descriptive title.
* Describe the exact steps which reproduce the problem.

### Suggesting Enhancements
* Check the existing issues to see if the enhancement has already been suggested.
* Open a new issue with a clear and descriptive title.
* Provide a step-by-step description of the suggested enhancement in detail.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!

## Styleguide

### JavaScript/TypeScript
* Use clean, readable code with descriptive variable names.
* Follow the existing project structure.
* We use ESLint for linting.

### CSS/Tailwind
* Use Tailwind CSS for all styling.
* Stick to the existing glassmorphism color palette.

## Community
* Please be respectful and cooperative.
* We follow a standard Code of Conduct (be kind!).

Happy coding!
